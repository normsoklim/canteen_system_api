Bakong QR Payment Test Guide

Overview

This guide explains how to test the Bakong QR code payment system for the canteen application. The system generates KHQR (Cambodian QR code standard) that is compatible with the Bakong mobile application.

QR Code Structure

The generated QR code follows the KHQR standard with the following structure:

  • Payload Format Indicator (00): 01 (Standard)
  • Point of Initiation Method (01): 12 (Dynamic)
  • Merchant Account Information (29): Contains Bakong-specific data
  • Transaction Currency (53): 840 (USD) or 116 (KHR)
  • Transaction Amount (54): Payment amount
  • Country Code (58): KH (Cambodia)
  • Additional Data Field (62): Reference number
  • CRC (63): 4-character checksum

Testing Steps

Step 1: Prepare Test Environment
  • Ensure your .env file has the correct Bakong configuration:
  • BAKONG_TOKEN=your_bakong_token
  • BAKONG_ACCOUNT_ID=your_bakong_account_id
  • BAKONG_MERCHANT_NAME=Your Merchant Name
  • BAKONG_MERCHANT_CITY=Phnom Penh
  • BAKONG_IS_TEST=true (for test environment)
Step 2: Access the Test Interface
  • Go to /bakong-payment for the main payment UI
  • Or go to /test-bakong-qr for the QR code testing interface
Step 3: Create a Test Order
  • If you don't have an existing order, create one through the API or database
  • Make sure the order has a valid ID and total amount
  • Example: Order ID = 1, Amount = $10.00
Step 4: Generate QR Code
  • Enter the order ID and amount in the test interface
  • Click "Generate QR Code"
  • The system will create a payment record and generate a KHQR code
  • The QR code will be displayed on the screen
Step 5: Test QR Code Validity
  • Use a QR code scanner app to scan the generated code
  • Verify that it contains the correct merchant information
  • Verify that it contains the correct amount
  • Verify that it follows the KHQR format
Step 6: Simulate Payment
  • Open the Bakong mobile app on your test device
  • Use the QR scan feature to scan the generated QR code
  • Complete the payment process in the Bakong app
  • Verify that the payment is processed correctly
Step 7: Verify Payment Status
  • Click "Check Payment Status" in the test interface
  • The system should update the payment status to "paid"
  • Check the database to confirm the payment record is updated

API Endpoints

Method Endpoint Description
POST /api/payments/bakong/initiate Initiate a Bakong payment and generate QR code
GET /api/payments/{id}/verify-bakong Check the status of a Bakong payment
POST /api/payments/verify-transaction Verify a transaction using MD5 hash
POST /api/payment/callback Handle payment callback from Bakong

Troubleshooting

  • QR code not generating: Check your Bakong configuration in .env file
  • Invalid QR format: Verify the KHQR library is properly installed and configured
  • Payment status not updating: Check the callback endpoint and payment verification logic
  • Test vs Production: Ensure BAKONG_IS_TEST is set correctly
  • Authentication errors: Make sure you have a valid JWT token for API access

Authentication

The Bakong payment UIs require authentication to access the API endpoints. To get an authentication token:

  1. Register a user via POST /api/register
  2. Login with credentials via POST /api/login
  3. Use the returned JWT token in the Authorization header as Bearer {token}
  4. Alternatively, store the token in localStorage as auth_token

For easier access, use the Authentication Helper page to register/login and automatically store your token.

Order Flow

The complete payment flow involves:

  1. Access the Menu Items page to select items
  2. Add items to your cart
  3. Proceed to checkout to create an order
  4. Use the Bakong payment UI to pay for your order
Important Notes:
  • This is a test environment - use test credentials only
  • For actual deployment, switch BAKONG_IS_TEST to false
  • Always validate payment status before fulfilling orders
  • Keep transaction references secure for verification purposes
