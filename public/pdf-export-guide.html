PDF Export Guide - Canteen API

This guide explains how to use the PDF export functionality in the Canteen API system.

Overview

The Canteen API provides functionality to export sales reports in PDF format. This feature allows administrators to generate and download comprehensive sales reports for specific date ranges.

Status: PDF export functionality is fully implemented and working correctly.

API Endpoint

GET /api/reports/export-pdf

Authentication

This endpoint requires authentication with a valid JWT token.

Include the token in the Authorization header:

Authorization: Bearer {your-jwt-token}

Required Role

User must have 'admin' role to access this endpoint.

Parameters

Parameter Type Required Description
start string (date) Yes Start date for the report in Y-m-d format (e.g., 2026-01-01)
end string (date) Yes End date for the report in Y-m-d format (e.g., 2026-12-31)

Example Usage

cURL Example

curl -X GET \ "http://localhost:8000/api/reports/export-pdf?start=2026-01-01&end=2026-12-31" \ -H "Authorization: Bearer your-jwt-token" \ -H "Accept: application/pdf" \ --output sales_report.pdf

JavaScript Example

// Get the JWT token from storage const token = localStorage.getItem('api_token') || sessionStorage.getItem('api_token'); // Define the date range const startDate = '2026-01-01'; const endDate = '2026-12-31'; // Make the API request fetch(`http://localhost:8000/api/reports/export-pdf?start=${startDate}&end=${endDate}`, { method: 'GET', headers: { 'Authorization': `Bearer ${token}`, 'Accept': 'application/pdf', } }) .then(response => { if (response.ok) { return response.blob(); } else { throw new Error('Export failed'); } }) .then(blob => { // Create a download link const url = window.URL.createObjectURL(blob); const a = document.createElement('a'); a.href = url; a.download = `sales_report_${startDate}_to_${endDate}.pdf`; document.body.appendChild(a); a.click(); window.URL.revokeObjectURL(url); a.remove(); }) .catch(error => { console.error('Export error:', error); });

Response

The API returns a PDF file with the following characteristics:

  • Content-Type: application/pdf
  • Content-Disposition: attachment; filename=sales_report_YYYY-MM-DD.pdf
  • File contains a formatted sales report with orders within the specified date range

Report Contents

The PDF report includes:

  • Report title and date range
  • Generated date and time
  • Table of orders with columns: ID, User ID, Total Amount, Status, Created At
  • Only completed orders are included in the report

Implementation Details

The PDF export functionality is implemented using the following components:

Backend Components

  • Controller: App\Http\Controllers\Api\ReportController - Contains the exportPdf method
  • Service: App\Services\ReportService - Provides the getOrdersForExport method
  • Repository: App\Repositories\ReportRepository - Contains the database query logic
  • View: resources/views/reports/sales.blade.php - HTML template for the PDF
  • Library: barryvdh/laravel-dompdf - PDF generation library

Configuration

The PDF library is configured in config/dompdf.php with appropriate settings for font handling and rendering.

Troubleshooting

Common Issues

  • 401 Unauthorized: Ensure you're sending a valid JWT token in the Authorization header
  • 403 Forbidden: Verify that the user account has admin role
  • 500 Server Error: Check that the date parameters are in the correct format (YYYY-MM-DD)
Important: When accessing the PDF export endpoint directly in a browser, you may see binary/text content instead of a downloaded PDF. This is normal browser behavior. To properly download the PDF, use the API with proper headers or use the test page provided at public/pdf-download-test.html.
Note: The PDF export functionality has been tested and confirmed to be working correctly. If you're experiencing issues, verify that your API server is running and that you have the correct authentication token.

Testing
